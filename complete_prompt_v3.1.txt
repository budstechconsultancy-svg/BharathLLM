ARCHITECTURE — READ BEFORE STARTING
================================================================================

THREE LAYERS:

  LAYER 1 — DATA SOURCES
  ┌─────────────────────────────────────────────────────────────────────┐
  │ Unstructured: PDFs (22 Indian langs) → OCR → Chunks → Qdrant RAG  │
  │ Structured:   PostgreSQL → 3-Tier Deterministic Text-to-SQL        │
  └─────────────────────────────────────────────────────────────────────┘

  LAYER 2 — INTELLIGENCE ENGINE
  ┌─────────────────────────────────────────────────────────────────────┐
  │ Query Classifier → routes to RAG, SQL, or MIXED                    │
  │ Fine-tuned LLaMA 3.1 8B reads fused multilingual context           │
  │ Result Fusion merges document chunks + database rows → answer      │
  └─────────────────────────────────────────────────────────────────────┘

  LAYER 3 — THREE PORTALS (React)
  ┌──────────────────┬──────────────────────┬──────────────────────────┐
  │ Dept/Min User    │ Dept/Min Admin        │ API Developer            │
  │ Chat interface   │ Upload + manage       │ Keys + code samples      │
  └──────────────────┴──────────────────────┴──────────────────────────┘

ISOLATION PRINCIPLE:
  Each Department (State) or Ministry (Central) → its own Qdrant collection.
  State:   {state_code}_{dept_slug}_docs  e.g. tn_health_docs, mh_finance_docs
  Central: cg_{ministry_slug}_docs        e.g. cg_health_ministry_docs
  API keys and JWT tokens are scoped to ONE collection. No cross-access.
  One shared fine-tuned LLM reads from any scoped collection.

WHY RAG + FINE-TUNING:
  Fine-tuning = the reader  (learns Indian govt writing style, terminology,
                              bilingual structure, script handling)
  RAG         = the library (provides actual document content at query time)

WHY RAG + SQL:
  RAG: "What does G.O. 142 say about hospital norms?"
  SQL: "How many CMHIS beneficiaries enrolled in 2024?"
  Mixed: "What is the scheme eligibility AND how many qualify now?"

WHY 3-TIER SQL (not LLaMA for SQL):
  LLaMA 3.1 8B is a document reader, not a code model.
  Wrong SQL against a live govt database is dangerous and unauditable.
  3-tier deterministic approach: keyword mapping → templates → safe fallback.

  SAKA_ERA_OFFSET = 78

  def normalise_numerals(text: str) -> str:
    for ind, ara in INDIAN_NUMERAL_MAP.items():
      text = text.replace(ind, ara)
    return text

  def convert_saka_date(saka_year: int, month: int, day: int) -> str:
    return f"{saka_year + SAKA_ERA_OFFSET}-{month:02d}-{day:02d}"

  def detect_scripts(text: str) -> dict:
    found = {}
    for code, info in INDIAN_LANGUAGES.items():
      for s, e in info["ranges"]:
        if any(s <= ord(c) <= e for c in text):
          found[code] = info["name"]; break
    return found

  def get_ocr_string() -> str:
    import os
    primary = os.getenv("PRIMARY_LANGUAGES","hi,en").split(",")
    codes = {INDIAN_LANGUAGES[l]["tess"] for l in primary if l in INDIAN_LANGUAGES}
    codes.update({"eng","hin"})
    return "+".join(sorted(codes))

  def get_collection_name(org_unit: str, state_code: str = None,
                          deployment_mode: str = "STATE_GOVT") -> str:
    slug = org_unit.lower().replace(" ","_").replace("&","and")
    if deployment_mode == "CENTRAL_GOVT":
      return f"cg_{slug}_docs"
    sc = (state_code or "xx").lower()
    return f"{sc}_{slug}_docs"

PROMPT 2.1 — MULTILINGUAL TEXT CLEANER + QUALITY SCORING

Build /pipeline/cleaner.py — government document cleaning for all Indian scripts.

CLEANING PIPELINE (apply in order):

1. BOILERPLATE REMOVAL:
   English/Hindi government headers:
   - "GOVERNMENT OF INDIA" | "GOVERNMENT OF [STATE]" as standalone line
   - "ABSTRACT" | "CONTENTS" | "TABLE OF CONTENTS" as standalone heading
   - Page numbers: "Page \d+ of \d+" | "- \d+ -" | "\[ \d+ \]"
   - Divider lines: 3+ consecutive dashes/underscores/equals on own line
   - Watermarks: "DRAFT" | "CONFIDENTIAL" | "COPY" as whole-line tokens
   - "Forwarded / Forward" as standalone line
   Hindi boilerplate: "भारत सरकार" | "राजपत्र" as repetitive standalone headers
   Other scripts: remove standalone header repetitions for each detected script.
   Preserve ALL Indian Unicode content otherwise.

2. OCR ERROR CORRECTION (only when source_type == "scanned"):
   English: broken hyphenated words across lines; missing space after period;
     "0" vs "O" in ref numbers (context rule: letter-preceded "0" → "O")
   Devanagari/Hindi: replace unrecognised glyphs (? □ ■) with [OCR_ERROR]
   South Indian scripts (Tamil/Telugu/Kannada/Malayalam): same placeholder
   Bengali/Odia/Gujarati/Punjabi: same
   Urdu/RTL: log RTL OCR errors separately — flag for manual review
   Log OCR error count per language per file.

3. NUMERAL NORMALISATION:
   Apply normalise_numerals() from language_registry.py
   Converts all Indian script numerals to ASCII digits for:
   - Date parsing consistency
   - Number extraction in SQL queries
   - Training data consistency

4. TERMINOLOGY STANDARDISATION:
   STATE mode: "G.O."|"GO"|"G O" → "Government Order"
     "Dt."|"dt."|"Dtd." → "Dated"; "Dept." → "Department"; "Rs." standardised
   CENTRAL mode additionally:
     "Govt. of India" → "Government of India"
     "MoF"|"Min. of Finance" → "Ministry of Finance"
     All ministry abbreviations → canonical names from metadata.json lookup
   Never normalise text in other scripts — English only.

5. WHITESPACE:
   Multiple spaces → single; 3+ blank lines → single blank; trailing spaces stripped.
   Preserve paragraph breaks (double newline). Preserve RTL markers for Urdu.

6. QUALITY SCORE (0–100):
   score_a = (words with 3–15 chars / total) × 100
   score_b = ocr_confidence (100 if text-based)
   score_c = min((sentence_count / (word_count / 20)) × 100, 100)
             Count sentence terminators: . ? ! । ॥ (including Devanagari danda)
   quality_score = (score_a × 0.4) + (score_b × 0.4) + (score_c × 0.2)

PROMPT 2.2 — SMART CHUNKER WITH AUTO-STRATEGY SELECTION
--------------------------------------------------------------------------------

Build /pipeline/chunker.py — multilingual document chunking for RAG.
Skip files with needs_review=True. Load metadata.json and manifest.json.

STRUCTURE SCORE (0–10):
  +3 if numbered sections > 3 (lines starting with "1."|"क."|"அ.")
  +3 if section markers > 2 ("ORDER:"|"CIRCULAR:"|"WHEREAS:"|"THEREFORE:"|
     "अनुभाग"|"विभाग"|"PART"|"SECTION"|"ANNEXURE"|"अनुलग्नक")
  +2 if paragraph breaks > 5
  +2 if scripts_present has any Indian script key (bias toward paragraph strategy)

STRATEGY SELECTION:
  score >= 6 AND only English/Hindi scripts → Strategy A (semantic/section)
  score 3–5                                 → Strategy B (fixed overlap)
  score < 3 OR multiple scripts present     → Strategy C (paragraph)

STRATEGY A — Section-based:
  Split on: ^\d+\. | ^[A-Z]+\. | ORDER: | CIRCULAR: | WHEREAS: | THEREFORE:
            PART [IVX]+ | SECTION \d+ | ANNEXURE | अनुभाग | विभाग
  Min 150 words, max 500 words. Overflow → Strategy B sub-chunking.
  Underflow (<150) → merge with next.

STRATEGY B — Fixed overlap:
  Window: 400 tokens (~300 words). Overlap: 50 tokens (~37 words).
  Sliding window over word list.

STRATEGY C — Paragraph:
  Split on \n\n. Merge if < 100 words. Cap at 500 using B sub-chunking.

CHUNK METADATA:

Print: docs/skipped, total chunks, strategy distribution, avg size, script breakdown.

RAGEngine class:

__init__(self, model_path=None):
  model_path defaults to MODELS_DIR/india-govt-llm-final/
  If missing: load base "meta-llama/Llama-3.1-8B-Instruct", log WARNING.
  Load model with 4-bit BitsAndBytesConfig (bfloat16, nf4, double_quant).
  Load tokenizer. Load BGE-M3. Connect Qdrant.
  Load /pipeline/collection_registry.json.
  Default top_k=5, score_threshold=0.35.

get_collection(self, org_unit, state_code):
  from pipeline.language_registry import get_collection_name
  return get_collection_name(org_unit, state_code, self.deployment_mode)

retrieve(self, query, org_unit, state_code, top_k=5, filters=None):
  Detect query language using ftlangdetect.
  Embed query with BGE-M3.
  collection = get_collection(org_unit, state_code)
  Build Qdrant filter from filters: doc_type, date_from, date_to, scripts.
  Search collection. Return results above score_threshold.
  If none above threshold: return [] with note.

generate(self, query, retrieved_chunks, query_lang="en"):
  If empty chunks:
    Return "This information is not available in the government documents."
  Build context with headers:
    "[Source: {original_file} | {doc_type} | {org_unit} | {date}]\n{text}\n---\n"
  System prompt:
    "You are an India Government document assistant.
     Answer ONLY from the provided government document context.
     Rules:
     1. Use only the provided context — no outside knowledge.
     2. If not in context: say exactly 'This information is not available
        in the provided government documents.'
     3. Cite source filename and date in every answer.
     4. Match the response language to the query language where possible.
        (Hindi query → Hindi answer; Tamil query → Tamil answer)
     5. Be concise, factual, and formal."
  Generate: temperature=0.0, do_sample=False, max_new_tokens=512,
    repetition_penalty=1.1

query(self, question, org_unit, state_code, filters=None):
  retrieve() → generate()
  confidence = avg of top-3 chunk scores (not all top_k)
  Return: {answer, sources, confidence, chunks_used, query_language}

SQLEngine class:

__init__(self):
  Connect to DATABASE_URL (SQLAlchemy, read-only user recommended).
  Load schema description from PostgreSQL information_schema at startup.
  Cache schema_map dict (see TIER 1).

TIER 1 — Keyword-to-column mapping (covers ~80% of government queries):

  SCHEMA_MAP = {
    "beneficiar":  ("scheme_enrollments","COUNT(*)","status='active'"),
    "enrolled":    ("scheme_enrollments","COUNT(*)",None),
    "enrolment":   ("scheme_enrollments","COUNT(*)",None),
    "scheme count":("scheme_enrollments","COUNT(*)",None),
    "budget":      ("budget_allocations","SUM(allocated_amount)",None),
    "allocation":  ("budget_allocations","SUM(allocated_amount)",None),
    "expenditure": ("budget_allocations","SUM(spent_amount)",None),
    "spent":       ("budget_allocations","SUM(spent_amount)",None),
    "utilisation": ("budget_allocations","SUM(spent_amount)/SUM(allocated_amount)*100",None),
    "balance":     ("budget_allocations","SUM(balance)",None),
    "fund":        ("budget_allocations","SUM(released_amount)",None),
    "employees":   ("employees","COUNT(*)","status='active'"),
    "staff count": ("employees","COUNT(*)","status='active'"),
    "vacancy":     ("employees","COUNT(*)","status='vacant'"),
    "hospitals":   ("hospitals","COUNT(*)",None),
    "beds":        ("hospitals","SUM(bed_count)",None),
    "doctors":     ("hospitals","SUM(doctor_count)",None),
    "nurses":      ("hospitals","SUM(nurse_count)",None),
    "schools":     ("schools","COUNT(*)",None),
    "students":    ("schools","SUM(student_count)",None),
    "teachers":    ("schools","SUM(teacher_count)",None),
    "land records":("land_records","COUNT(*)",None),
    "patta":       ("land_records","COUNT(*)",None),
  }

  Also map common Hindi/regional query keywords:
  {
    "लाभार्थी":   ("scheme_enrollments","COUNT(*)",None),  # Hindi: beneficiary
    "बजट":        ("budget_allocations","SUM(allocated_amount)",None),  # budget
    "कर्मचारी":  ("employees","COUNT(*)","status='active'"),  # employees
    "अस्पताल":   ("hospitals","COUNT(*)",None),  # hospitals
    "विद्यालय":  ("schools","COUNT(*)",None),  # schools
    "பயனாளர்":   ("scheme_enrollments","COUNT(*)",None),  # Tamil: beneficiary
    "பள்ளி":     ("schools","COUNT(*)",None),  # Tamil: school
    "ஆசிரியர்": ("schools","SUM(teacher_count)",None),  # Tamil: teacher
  }

  generate_sql_tier1(question, org_unit, state_code):
    q_lower = question.lower()
    # Also normalise numerals in question for year/number extraction
    q_norm = normalise_numerals(question)
    matched = None
    for keyword, (table, agg, extra) in SCHEMA_MAP.items():
      if keyword in q_lower or keyword in question:  # check original for non-ASCII
        matched = (table, agg, extra); break
    if not matched: return None

    table, agg, extra = matched
    where = [f"state_code='{state_code}'"] if state_code else []
    col = "department" if deployment_mode == "STATE_GOVT" else "ministry"
    if org_unit != "Unknown": where.append(f"{col}='{org_unit}'")
    if extra: where.append(extra)

    year = re.search(r'\b(20\d{2}|19\d{2})\b', q_norm)
    if year: where.append(f"year={year.group(1)}")

    district = re.search(r'\bin\s+([A-Za-z\u0900-\u0D7F]+)\b', question, re.I)
    if district: where.append(f"district='{district.group(1).title()}'")

    where_str = " AND ".join(where) if where else "1=1"
    return f"SELECT {agg} as result FROM {table} WHERE {where_str} LIMIT 100"

TIER 2 — Template fill (for structured multi-column queries):

  TEMPLATES = {
    "list":         "SELECT * FROM {table} WHERE {where} ORDER BY created_at DESC LIMIT 20",
    "by_district":  "SELECT district,COUNT(*) as cnt FROM {table} WHERE {where} GROUP BY district ORDER BY cnt DESC LIMIT 25",
    "by_year":      "SELECT year,{agg} as total FROM {table} WHERE {where} GROUP BY year ORDER BY year DESC LIMIT 10",
    "by_type":      "SELECT type,COUNT(*) as cnt FROM {table} WHERE {where} GROUP BY type ORDER BY cnt DESC LIMIT 20",
    "top_schemes":  "SELECT scheme_name,COUNT(*) as cnt FROM scheme_enrollments WHERE {where} GROUP BY scheme_name ORDER BY cnt DESC LIMIT 10",
  }

  generate_sql_tier2(question, org_unit, state_code):
    q_lower = question.lower()
    # Detect template type
    if any(k in q_lower for k in ["list","show all","get all","display"]): tmpl="list"
    elif any(k in q_lower for k in ["district","districtwise","district-wise"]): tmpl="by_district"
    elif any(k in q_lower for k in ["year","yearly","annual","financial year"]): tmpl="by_year"
    elif any(k in q_lower for k in ["type","category","kind"]): tmpl="by_type"
    elif any(k in q_lower for k in ["scheme","top scheme","popular"]): tmpl="top_schemes"
    else: return None

    # Determine table from keyword scan
    for keyword,(table,agg,_) in SCHEMA_MAP.items():
      if keyword in q_lower: break
    else: table = "scheme_enrollments"; agg = "COUNT(*)"

    col = "department" if os.getenv("DEPLOYMENT_MODE") == "STATE_GOVT" else "ministry"
    where_parts = []
    if state_code: where_parts.append(f"state_code='{state_code}'")
    if org_unit != "Unknown": where_parts.append(f"{col}='{org_unit}'")
    where = " AND ".join(where_parts) if where_parts else "1=1"
    return TEMPLATES[tmpl].format(table=table, where=where, agg=agg)

TIER 3 — Safe fallback (no LLaMA, no guessing):
  Return {sql_generated:None, rows:[], formatted_result:
    "This query requires live database access. Please contact your
     department data team or check the department dashboard directly.",
    error:"Query pattern not matched"}

validate_sql(self, sql):
  Reject if contains (case-insensitive): DROP DELETE UPDATE INSERT ALTER TRUNCATE
  Reject if not starting with SELECT
  Auto-add LIMIT 100 if no LIMIT clause
  Return sanitised SQL or None

execute_sql(self, sql):
  Validate first. Execute on read-only connection.
  Catch all exceptions → return {error, rows:[], column_names:[]}
  Return: {rows, column_names, row_count}

format_results(self, question, rows, column_names):
  Empty: "No records found in the database for this query."
  Single value: return as string
  <= 10 rows: readable table text
  > 10 rows: "Found {n} records. Showing first 10:\n" + first 10

query(self, question, org_unit, state_code):
  Tier1 → Tier2 → Tier3
  Return: {sql_generated, rows, formatted_result, error, row_count}

    "G.O.","government order","circular","notification","scheme guidelines",
    "policy","eligibility criteria","mentioned in","stated in","what rule",
    "under which section","when was","how to apply",
    # Hindi:
    "क्या कहता है","नियम","प्रक्रिया","पात्रता","सरकारी आदेश",
    # Tamil:
    "என்ன சொல்கிறது","விதிமுறை","நடைமுறை","தகுதி","அரசாணை",

    "ఏమి చెప్పింది","నిబంధన","విధానం","అర్హత",
    # Kannada:
    "ಏನು ಹೇಳುತ್ತದೆ","ನಿಯಮ","ಪ್ರಕ್ರಿಯೆ","ಅರ್ಹತೆ",
  ]

  classify(question):
    has_sql = any(k in question.lower() or k in question for k in SQL_KEYWORDS)
    has_rag = any(k in question.lower() or k in question for k in RAG_KEYWORDS)
    if has_sql and has_rag: return "MIXED"
    if has_sql: return "SQL"
    if has_rag: return "RAG"
    return "RAG"  # default

           confidence, chunks_used, query_language}

Create /auth/auth_service.py — functions:

  hash_password(plain) → bcrypt; verify_password(plain, hashed) → bool

  create_user(employee_id, email, password, full_name, org_unit,
              state_code, role, deployment_mode):
    Insert User row. Hash password. must_change_password=True. Return user dict.

  authenticate_user(employee_id_or_email, password, db):
    Lookup by employee_id OR email. verify_password(). Update last_login.
    Return user or raise HTTPException 401.

  create_access_token(user_id, org_unit, state_code, role, deployment_mode):
    JWT payload: {sub:user_id, org:org_unit, state:state_code,
                  role:role, mode:deployment_mode, exp:now+JWT_EXPIRY_HOURS}
    Sign with JWT_SECRET_KEY + JWT_ALGORITHM. Return token string.

  verify_token(token, db):
    Decode JWT. Check expiry. Check sessions table not revoked.
    Return payload dict or raise 401.

  create_api_key(name, org_unit, state_code, deployment_mode,
                 created_by_id, rate_limit, expires_at, db):
    raw_key = "sk-ig-" + {state_code[:2].lower()} + "-" + secrets.token_hex(16)
    Store SHA-256(raw_key) in api_keys table. Return raw_key ONCE.

  verify_api_key(raw_key, db):
    SHA-256 hash → lookup. Check is_active + expires_at. Update last_used_at.
    Return api_key record.

  change_password(user_id, current_password, new_password, db):
    Verify current. New must: len>=12, upper+lower+digit+symbol, != current.
    Hash new. UPDATE users. REVOKE all sessions for user.
    Return True or raise 400.

Create /auth/dependencies.py:
  get_current_user() — Bearer JWT header → verify_token() → user context
  get_api_key_context() — X-API-Key header → verify_api_key() → context
  unified auth dependency: tries JWT first, falls back to X-API-Key.
  Both return: {user_id_or_key_hash, org_unit, state_code,
                deployment_mode, role, rate_limit}
  require_role(roles: list) — factory dependency, raises 403 if not in roles.

Create /auth/rate_limiter.py:

    Redis INCR "rate:{key}"; set TTL on first hit; raise 429 if over limit.

PROMPT 7.2 — ACCURACY BENCHMARKING + MULTILINGUAL REPORTING
--------------------------------------------------------------------------------

Build /evaluation/benchmark.py — comprehensive accuracy measurement.

PREREQUISITE: /evaluation/eval_questions.json must have >= 50 questions.

METRICS:

  M1 — Retrieval Accuracy (RAG questions only):
    Retrieval@1, @3, @5 — did source_file appear in top N results?

  M2 — Answer Faithfulness:
    Content word overlap between answer and retrieved chunks.
    Remove stopwords (load from indian_stopwords.json by question_language).
    Score: 0.0–1.0

  M3 — Answer Relevance (semantic):
    Cosine similarity: embed(question) vs embed(answer) via BGE-M3.
    Score: 0.0–1.0

  M4 — Hallucination Detection:
    Per sentence in answer: max cosine similarity to any retrieved chunk.
    Flag if max_similarity < 0.35. Rate = flagged/total sentences.

  M5 — SQL Accuracy (SQL-type questions):
    Valid SQL generated? Rows returned? Answer contains expected value?
    Score: 0.0–1.0

  M6 — Multilingual Accuracy:
    Group by question_language. Report all metrics per language.
    Detect if model degrades on specific scripts.

  M7 — Latency: p50, p90, p95 ms.

PASS THRESHOLDS:
  Retrieval@5 > 85%, Faithfulness > 80%, Hallucination < 5%,
  p95 < 10,000ms, SQL accuracy > 80% (if SQL qs present).
  Per-language: warn if any language Retrieval@5 < 70%.
