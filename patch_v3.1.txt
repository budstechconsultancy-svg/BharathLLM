DEPLOYMENT CONFIGURATION — SET ONCE BEFORE ANY PROMPT
--------------------------------------------------------------------------------

Before running Prompt 1.1, set these two configuration values in your .env.
Every subsequent prompt reads these values to adapt its behaviour.

DEPLOYMENT_MODE=
  Options:
    CENTRAL_GOVT       — Central Government of India (Ministries, Acts, Gazettes)
    STATE_GOVT         — State Government (Departments, G.O.s, Circulars)

DEPLOYMENT_STATE=
  Required only when DEPLOYMENT_MODE=STATE_GOVT
  Options (ISO 3166-2:IN state codes):
    TN    — Tamil Nadu      (primary language: Tamil)
    MH    — Maharashtra     (primary language: Marathi)
    KA    — Karnataka       (primary language: Kannada)
    KL    — Kerala          (primary language: Malayalam)
    AP    — Andhra Pradesh  (primary language: Telugu)
    TS    — Telangana       (primary language: Telugu)
    WB    — West Bengal     (primary language: Bengali)
    GJ    — Gujarat         (primary language: Gujarati)
    PB    — Punjab          (primary language: Punjabi)
    OD    — Odisha          (primary language: Odia)
    AS    — Assam           (primary language: Assamese)
    UP    — Uttar Pradesh   (primary language: Hindi)
    RJ    — Rajasthan       (primary language: Hindi)
    HR    — Haryana         (primary language: Hindi)
    MP    — Madhya Pradesh  (primary language: Hindi)
    BR    — Bihar           (primary language: Hindi)
    JH    — Jharkhand       (primary language: Hindi)
    CG    — Chhattisgarh    (primary language: Hindi)
    UK    — Uttarakhand     (primary language: Hindi)
    HP    — Himachal Pradesh (primary language: Hindi)
    GA    — Goa             (primary language: Konkani)
    MN    — Manipur         (primary language: Manipuri/Meitei)
    Other states: follow same pattern

PRIMARY_LANGUAGES=
  Comma-separated list of language codes for this deployment.
  These receive priority in OCR, cleaning, and training.
  Central Govt example:  PRIMARY_LANGUAGES=hi,en
  Tamil Nadu example:    PRIMARY_LANGUAGES=ta,en,hi
  Karnataka example:     PRIMARY_LANGUAGES=kn,en,hi
  Kerala example:        PRIMARY_LANGUAGES=ml,en,hi
  West Bengal example:   PRIMARY_LANGUAGES=bn,en,hi
  Always include 'en' and 'hi' for Central Govt documents.

WHAT CHANGES BASED ON DEPLOYMENT_MODE:
  CENTRAL_GOVT:
    - Document types: Gazette Notification, Act, Bill, Ministry Order,
      Cabinet Note, Policy Document, SOP, Advisory, Circular, Notification
    - Organisational units: Ministries (not Departments)
    - Primary languages: Hindi + English + state languages of relevant offices
    - Qdrant collections: tn_{ministry_slug}_docs
      e.g. tn_finance_ministry_docs, tn_health_ministry_docs
    - Date format includes "Saka Era" dates found in Gazette documents
    - Reference format: F.No., D.O. No., O.M. No., S.O., G.S.R.

  STATE_GOVT:
    - Document types: G.O., Circular, Scheme Guidelines, Notification,
      Proceedings, Tender, Gazette (State), Policy
    - Organisational units: Departments
    - Primary languages: State language + Hindi + English
    - Qdrant collections: {state_code}_{dept_slug}_docs
      e.g. tn_health_docs, mh_education_docs, ka_finance_docs
    - Reference format: G.O. Ms., G.O. Rt., Proc. No., Circular No.

  def get_deployment_languages(deployment_state: str, deployment_mode: str) -> list:
    """
    Returns ordered list of language codes for a given deployment.
    Priority: state primary language > Hindi > English > other Indian languages
    """
    import os
    primary = os.getenv("PRIMARY_LANGUAGES", "hi,en").split(",")
    all_langs = list(INDIAN_LANGUAGES.keys())
    # Put primary languages first, then rest
    ordered = primary + [l for l in all_langs if l not in primary]
    return ordered

  def get_ocr_lang_string(deployment_state: str) -> str:
    """
    Returns tesseract lang string for this deployment.
    e.g. "tam+hin+eng" for Tamil Nadu, "hin+eng" for Central Govt
    """
    import os
    primary = os.getenv("PRIMARY_LANGUAGES", "hi,en").split(",")
    # Always include primary + hindi + english as minimum
    codes = set()
    for lang_code in primary:
      if lang_code in INDIAN_LANGUAGES:
        codes.add(INDIAN_LANGUAGES[lang_code]["tesseract_code"])
    # Always ensure eng and hin are present
    codes.add("eng")
    codes.add("hin")
    return "+".join(sorted(codes))

  def detect_scripts_present(text: str) -> dict:
    """
    Scan text and return which Indian scripts are present.
    Returns: {lang_code: True/False}
    """
    results = {}
    for lang_code, info in INDIAN_LANGUAGES.items():
      for start, end in info["unicode_ranges"]:
        if any(start <= ord(c) <= end for c in text):
          results[lang_code] = True
          break
      else:
        results[lang_code] = False
    return {k: v for k, v in results.items() if v}  # only present scripts

    "staff": ("employees", "COUNT(*)", "status='active'"),
    "hospitals": ("hospitals", "COUNT(*)", None),
    "beds": ("hospitals", "SUM(bed_count)", None),
    "doctors": ("hospitals", "SUM(doctor_count)", None),
    "schools": ("schools", "COUNT(*)", None),
    "students": ("schools", "SUM(student_count)", None),
    "teachers": ("schools", "SUM(teacher_count)", None),
  }

  generate_sql_tier1(question, department):
    question_lower = question.lower()
    matched = []
    for keyword, (table, agg, extra_where) in SCHEMA_MAP.items():
      if keyword in question_lower:
        matched.append((table, agg, extra_where))
    if not matched: return None  # fall through to Tier 2

    table, agg, extra_where = matched[0]
    where_clauses = [f"department='{department}'"]
    if extra_where: where_clauses.append(extra_where)

    # Extract year/district from question using regex + numeral normaliser
    year_match = re.search(r'\b(20\d{2})\b', normalise_numerals(question))
    if year_match: where_clauses.append(f"year={year_match.group(1)}")
    district_match = re.search(
      r'\bin\s+([A-Za-z\u0900-\u0D7F]+)\b', question, re.IGNORECASE)
    if district_match:
      where_clauses.append(f"district='{district_match.group(1).title()}'")

    where = " AND ".join(where_clauses)
    sql = f"SELECT {agg} as result FROM {table} WHERE {where} LIMIT 100"
    return sql

TIER 2 — Schema-aware template fill (for structured queries):
  If Tier 1 returns None, use a fast template approach:

  QUERY_TEMPLATES = {
    "list_by_dept":
      "SELECT * FROM {table} WHERE department='{dept}' LIMIT {limit}",
    "count_by_district":
      "SELECT district, COUNT(*) as count FROM {table}
       WHERE department='{dept}' GROUP BY district ORDER BY count DESC LIMIT 20",
    "sum_by_year":
      "SELECT year, SUM({amount_col}) as total FROM {table}
       WHERE department='{dept}' GROUP BY year ORDER BY year DESC LIMIT 5",
  }

  Detect template type using keyword patterns:
    "list" | "show all" | "get all" → list_by_dept
    "by district" | "district-wise" | "districtwise" → count_by_district
    "by year" | "year-wise" | "financial year" → sum_by_year
  Fill template with detected table and department. Return SQL.

TIER 3 — Fallback: structured error response (NOT LLaMA):
  If Tier 1 and Tier 2 both fail:
  Return: {sql_generated: null, rows: [], formatted_result:
    "This query requires accessing live database records. Please contact
     your department's data team or use the department portal directly.",
    error: "Query too complex for automatic SQL generation"}

VALIDATE ALL TIERS through the same validate_sql() function (unchanged).

WHY NO LLaMA FOR SQL:
  LLaMA 3.1 8B is a document-reading model, not a code model. Text-to-SQL
  requires a model fine-tuned on SQL (like sqlcoder or defog/sqlcoder-7b).
  For a government deployment, incorrect SQL against a live database is
  dangerous. The three-tier approach is deterministic and auditable.

  Future upgrade path: when the project has a SQL-specific model budget,
  deploy defog/sqlcoder-7b-hf as a separate inference endpoint and replace
  generate_sql_tier1/2 with a single API call to that endpoint.
